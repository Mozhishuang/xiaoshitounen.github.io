一.导语

Mac 平台一些 IDA 集成开发工具的安装

准备：

  • 微信公众号：Mac软件管家，获取一些软件安装包。
  • 开启“应用允许任何来源”，终端输入 sudo spctl --master-disable 命令，输入密码回车即可。
  • 已经勾选允许任何来源，打开安装好的程序提示“文件已损坏”，终端输入 sudo xattr -r -d com.apple.quarantine 程序位置 。例如，Sketch的命令是 sudo xattr -r -d com.apple.quarantine /Applications/Sketch.app。程序位置可以通过将应用程序目录中的程序拖入终端得到。

②. 问题出现了，终端输入conda命令无法识别。

  • 使用命令vi ~/.bash_profile，在vi下添加命令export PATH="/Users/nick(你自己的安装路径)/anaconda3/bin:$PATH"，然后保存退出。关于vi的操作可以参考这篇文章vim（vi）下的三种模式及其相关命令。

  • 退出后，输入命令 source $HOME/.bash_profile ，达到立即生效的效果。

  • 输入命令 source $HOME/.bash_profile ，达到立即生效的效果。

  • 输入命令 echo $PATH，验证是否加入成功。

  • 输入命令 conda --version，测试是否成功，结果是可以的。

③. 问题并没有结束，关闭终端再打开conda命令就无法识别了？但是执行 source ~/.bash_profile 之后就可以了。终端默认 bash 我换成了 zsh。因为 zsh 加载的是 ~/.zshrc 文件，而 .zshrc 文件中并没有定义任务环境变量。解决办法在 ~/.zshrc 文件最后，增加一行：source ~/.bash_profile 。但是每次打开终端前面都有一个（base），很难受。使用命令 conda config --set auto_activate_base false 解决问题。
